## 3. Blueprint Must Explicitly Pull Engine Outputs

Please confirm and implement the following:

1. Blueprint must explicitly read outputs from each engine result object.
2. Each blueprint field must be mapped to its corresponding engine output.
3. If an engine output exists but the field is empty, the extractor must populate it.
4. Blueprint generation must not default to manual mode unless the engine actually returned empty output.

This is likely an extraction / mapping issue rather than a generation issue.
