Crawler artifacts are entering the analysis layer and corrupting the signals.

Examples detected:

* `[synthetic-positive-*]`
* `engagement signal`
* `reported comments`

These are **crawler artifacts**, not real audience language.

Because they are entering the analysis pipeline they produce:

* false objections
* fake pain signals
* incorrect audience segments

### Required Action

Create a **Signal Sanitation Layer** before ANY analysis occurs.

Before deterministic modules run, filter out any text containing:

* `synthetic`
* `engagement signal`
* `reported comments`
* `[synthetic-*]`

No sanitized signals → **analysis must not run**.

This filter must sit **between the crawler ingestion layer and the Audience Engine pipeline.**

Before AI segmentation runs:
