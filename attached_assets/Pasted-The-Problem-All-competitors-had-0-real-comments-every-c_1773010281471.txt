The Problem: All competitors had 0 real comments — every comment was synthetic. Three interconnected issues were causing this:

Bright Data Web Unlocker doesn't support sticky sessions (port 33335). The proxy pool was appending -session-xxx to the username, causing every proxy request to fail with "Request was cancelled." This meant all scraping through the proxy failed with duration=0ms.
Instagram locked down comment APIs. The web_profile_info API returns post data but no longer includes comment text. The GraphQL comment endpoint returns 401 (requires login). The V1 post endpoint returns 404. All three existing comment paths were dead.
Missing Sec-Fetch headers on several request paths (i.instagram.com, feed pagination, GraphQL comments) caused Bright Data to reject those requests with "SecFetch Policy violation."
The Fixes:

Proxy fix: createSession() now skips the -session- suffix when using Web Unlocker (port 33335), so proxy connections work correctly.
New HTML comment scraper: Added scrapePostCommentsFromHTML() that fetches the full post HTML page and extracts preview_comments from the embedded JSON. This reliably returns 2-3 real comments per post.
Complete fallback chain: GraphQL (any failure) → V1 API (any failure) → HTML scrape. Every stage now falls through to the next.
Sec-Fetch headers added to all Instagram request paths.
Result: Real comments are now being scraped successfully — verified with live tests showing ~2 real comments per post from 5 different posts. All 368 tests continue to pass.
