Hi Team,

We need to introduce a new foundational layer inside BuildUp Plan called Content DNA.

This is an important product architecture update, not a minor enhancement.

Right now, the plan is able to produce strategic sections such as content distribution, KPI monitoring, creative testing, budget allocation, and risk triggers. This is good, but it is still missing one critical layer:

The plan can tell the user what to publish, but it still does not fully teach the user how the content itself should be built.

For MarketMind to become a true AI marketing operating system, BuildUp Plan must not stop at strategy outputs. It must also synthesize all relevant engine analysis into a usable content operating blueprint.

---

## 1. Introduce a New Layer: Content DNA

Please add a structured layer inside BuildUp Plan called:

Content DNA

This layer must become the system’s single source of truth for how content should be created for the active business, offer, audience, and market context.

Content DNA must be generated by synthesizing the outputs of all relevant engines, especially:

- Market Intelligence
- Audience Engine
- Positioning Engine
- Differentiation Engine
- Offer Engine
- Funnel Engine
- Awareness Engine
- Persuasion Engine
- Channel Selection
- Iteration learnings where available

This means Content DNA is not guessed manually and not generated from generic AI defaults.  
It must be built from actual system analysis.

---

## 2. Purpose of Content DNA

Content DNA must answer questions such as:

- What kind of messaging style should this business use?
- What kind of hooks fit this market?
- What kind of CTA should be used?
- What narrative structure should content follow?
- What content angle should dominate?
- What visual direction fits the offer and audience?
- What format style is preferred?
- How direct or soft should the persuasion be?
- What content should build trust versus convert?
- What must be avoided because it conflicts with market reality or positioning?

This is critical because content is the execution core of the system.

The user should not only receive a plan.  
The user should receive a content blueprint that explains how to create content correctly.

---

## 3. Content DNA Must Be Built on Real Inputs

Content DNA must follow this rule:

Use the real business offer defined in the Profile Card as the primary business truth.  
The system may optimize or refine the offer using engine logic, but it must not disconnect from the actual offer the user entered.

In other words:

- Profile Card offer = source truth
- Engine analysis = optimization and strategic shaping
- Content DNA = practical content behavior derived from both

This prevents strategy drift and ensures the content remains usable in the real business context.

---

## 4. Required Structure of Content DNA

Please structure Content DNA into clear sections. At minimum, it must contain:

### A. Messaging Core
- primary value promise
- positioning emphasis
- tone style
- trust style
- persuasion intensity

### B. CTA DNA
- primary CTA type
- CTA delivery style
- soft vs direct CTA rule
- CTA examples that fit the plan

### C. Hook DNA
- preferred hook types
- hook intensity
- recommended opening style
- recommended hook duration
- hook examples aligned with audience and offer

### D. Narrative DNA
- preferred narrative structure
- problem/solution balance
- trust-building sequence
- authority vs relatability ratio
- storytelling guidance

### E. Content Angle DNA
- dominant angles to use
- angles to avoid
- educational vs proof vs offer ratio
- trust-building content logic

### F. Visual DNA
- visual direction
- talking-head vs proof-based vs screen/demo preference
- static vs dynamic preference
- simplicity vs polished-production preference

### G. Format DNA
- reel behavior
- carousel behavior
- story behavior
- post behavior
- when each format should be used

### H. Execution Rules
- what content must always include
- what content must never do
- how content should support funnel movement
- how content should support the real offer

This must be practical, readable, and executable.

---

## 5. Content DNA Must Feed the Agent

This is a critical requirement.

The MarketMind Agent must have read access to Content DNA and must be able to use it in three ways:

### A. Explain It
If the user asks:
- explain the hook strategy
- explain the CTA style
- explain how I should structure reels
- explain why this tone is recommended

The agent must answer from Content DNA, not from generic AI assumptions.

### B. Guide Execution
If the user asks:
- what should I film today?
- how should this reel start?
- what hook should I use?
- what CTA should I end with?

The agent must use Content DNA and current plan context.

### C. Support Creation
When the user sends work into Creation, the agent should be able to pass along Content DNA rules so the generated content matches the system strategy.

---

## 6. Add Content DNA Summary to the Agent Box

Please also update the Agent Box so that it surfaces a compact operational summary of Content DNA.

The Agent Box should include a small “Content DNA Snapshot” area that can show items like:

- CTA type
- hook style
- recommended hook duration
- narrative style
- content angle
- format preference

Example:

Content DNA  
CTA: Soft WhatsApp CTA  
Hooks: Problem-led + authority hooks  
Hook length: 1–3 seconds  
Narrative: Problem → insight → proof → soft CTA  
Format priority: Reels first

This must be derived from the plan and engine synthesis, not typed manually.

The goal is that the user can look at the Agent Box and immediately understand:

How should I make content for this business?

---

## 7. BuildUp Plan Must Become the Source of Content Behavior

BuildUp Plan should not only output:
- content distribution
- budget
- testing
- KPI tracking

It must also output:
- the behavioral rules of the content itself

That is what Content DNA is.

So the architecture becomes:

Engines  
→ Orchestrator  
→ BuildUp Plan  
→ Content DNA  
→ Agent / Creation / Calendar / Execution

This keeps the system aligned and prevents disconnected content production.

---

## 8. Acceptance Criteria

Please consider this complete only if all of the following are true:

- Content DNA exists as a first-class layer inside BuildUp Plan
- Content DNA is built from engine outputs, not generic fallback text
- Content DNA uses the real offer from the Profile Card as its business anchor
- Agent can explain Content DNA sections individually
- Agent can use Content DNA to guide content execution
- Agent Box shows a compact Content DNA summary
- Creation can consume Content DNA guidance when generating assets

This is an important upgrade because content is the real execution core of MarketMind.  
Without this layer, the system can tell the user what to publish, but not fully teach them how to make the content correctly.

Please implement this with depth and precision.

Thank you.
