- posts_processed  
- competitors_processed  
- signals_detected  
- signals_filtered

## 2. Expand Signal Pattern Detection

Current signal detection appears too narrow. Signals must be detected from multiple semantic categories, not just pain statements.

Expand extraction patterns to include:

- pain signals  
- desire signals  
- transformation statements  
- authority positioning claims  
- differentiation narratives  
- competitor weaknesses  
- audience objections  
- strategic claims  

Signals should be detected even when phrasing differs.

---

## 3. Replace Binary Filtering with Weighted Scoring

Current filtering likely removes many signals.

Instead of:

signal_strength > threshold

Use a weighted scoring model:

signal_score =
frequency_weight +
semantic_strength +
cross-competitor_confirmation

Signals should only be removed if the score is extremely weak.

## 4. Cross-Competitor Signal Aggregation

Signals should be reinforced when multiple competitors express similar themes.

Example:

If 4 competitors express similar positioning narratives, treat this as a clustered signal rather than separate weak signals.

Implement basic clustering so the system recognizes repeated market narratives.

---

## 5. Signal Density Safeguard Adjustment

Signal density warnings should trigger only when both of the following are true:

- competitors analyzed < minimum threshold  
- signals_detected < minimum threshold

- competitors analyzed
