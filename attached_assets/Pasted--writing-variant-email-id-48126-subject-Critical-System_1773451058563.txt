Replit Engineering Team,

We need to implement a structural upgrade across the strategy engines to resolve several systemic weaknesses identified during runtime analysis. These updates must be implemented with strict validation and with safeguards against common regression issues previously observed in the system.

This is not a cosmetic change. The goal is to bring the system to production-grade strategic reliability (10/10) while preserving the integrity of the Master Plan architecture and engine isolation rules.

Below are the required improvements.

⸻

1. Funnel Completion Enforcement

Current Issue
The Channel Selection Engine sometimes produces strategies without a clear conversion layer, resulting in incomplete funnels (Awareness + Nurture without Conversion).

Required Implementation
Add mandatory funnel validation inside the Channel Selection Engine and Funnel Resolution Layer.

Rules:
	•	Every strategy must contain at least one channel mapped to each stage:
	•	Awareness
	•	Nurture
	•	Conversion
	•	If the system cannot find a valid conversion channel, it must:
	•	Trigger funnel reconstruction
	•	Inject an appropriate conversion channel (Search Ads, Landing Conversion Path, Offer Page, etc.)

Acceptance Criteria
	•	Engine must reject strategies missing any funnel stage.
	•	Funnel reconstruction must auto-assign roles when possible.
	•	A conversionChannelAssigned flag must exist in the output payload.
	•	Decision Gate must validate the final funnel structure.

⸻

2. Persuasion Compatibility Optimization

Current Issue
Persuasion compatibility scores are dropping too low, causing unnecessary channel downgrades.

Required Implementation
Enhance the Persuasion Engine compatibility model to better evaluate:
	•	persuasionMode
	•	channelBehavior
	•	funnelRole
	•	content format compatibility

Additionally, introduce a persuasion correction mechanism:

If persuasion compatibility < 50%:
	•	Attempt persuasion adjustment
	•	Recalculate channel role suitability
	•	Only reject the channel if all roles fail

Acceptance Criteria
	•	Persuasion compatibility must never block a channel before funnel reconstruction is attempted.
	•	Post-reconstruction validation must run again via Decision Gate.
	•	All persuasion adjustments must appear in the reconstruction log.

⸻

3. Retention Engine Data Model Upgrade

Current Issue
Retention Engine is operating in fallback mode due to missing customer metrics.

Required Implementation
Create a Manual Retention Metrics Input Model so the engine can operate without external integrations.

Add a new schema:

RetentionMetrics:
	•	repeatPurchaseRate
	•	averageOrderValue
	•	customerLifespan
	•	refundRate
	•	purchaseFrequency
	•	monthlyCustomers

These values must be available through a manual input interface and accessible by the Retention Engine.

Acceptance Criteria
	•	If metrics exist → run full retention analysis.
	•	If metrics are missing → fallback mode activates but must still generate baseline loops.
	•	Retention Engine must calculate:
	•	estimated LTV
	•	churn risk
	•	retention strength score.

⸻

4. Organic vs Paid Channel Mode System

Current Issue
Channel selection sometimes defaults excessively to organic channels.

Required Implementation
Add a Channel Mode Selector inside the Build a Plan section.

Modes:
	•	Organic Only
	•	Paid Only
	•	Hybrid
	•	Automatic (AI chooses optimal strategy)

Behavior:

If Organic:
	•	Only organic channels allowed.

If Paid:
	•	Only paid channels allowed.

If Hybrid:
	•	Mix allowed with budget distribution logic.

If Automatic:
	•	The engine selects the most strategic mix based on:
	•	audience density
	•	budget constraints
	•	persuasion compatibility
	•	cost efficiency

Acceptance Criteria
	•	Channel mode must be passed into the Channel Selection Engine.
	•	Engine must filter candidate channels according to the mode.
	•	Automatic mode must be logged with reasoning metadata.

⸻

5. Decision Gate Strengthening

Current Issue
Decision Gate confidence scoring is occasionally too permissive or too restrictive.

Required Implementation
Refine Decision Gate scoring logic to include:
	•	funnel integrity score
	•	persuasion alignment score
	•	budget realism
	•	channel scalability

Acceptance Criteria
	•	Decision Gate must reject structurally invalid strategies.
	•	Decision Gate must run both:
	•	pre-reconstruction
	•	post-reconstruction validation.

⸻

This must stop.
