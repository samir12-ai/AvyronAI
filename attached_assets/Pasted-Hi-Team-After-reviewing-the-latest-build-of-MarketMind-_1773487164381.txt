# 1. Orchestrator Not Triggering Full Engine Generation

The first issue is that engine generation is not being executed automatically from the orchestration entry point.

The intended architecture is:

Market Intelligence  
→ Audience Engine  
→ Positioning  
→ Differentiation  
→ Offer Engine  
→ Funnel Engine  
→ Integrity Engine  
→ Awareness Engine  
→ Persuasion Engine  
→ Statistical Validation  
→ Budget Governor  
→ Channel Selection  
→ Iteration Engine  
→ Retention Engine  
→ Plan Synthesis

Currently, the system does not appear to trigger all engines sequentially through the orchestrator.

Instead, the behavior suggests:

• Only partial engine execution is occurring  
• Some engines are never triggered  
• The pipeline UI shows engines, but they are not all executed programmatically

Required fix:

The Run Analysis / Plan Generation trigger must call the orchestrator, and the orchestrator must execute every engine sequentially with proper context propagation.

The orchestrator must:

• Trigger each engine in the defined order  
• Pass structured context between engines  
• Persist outputs after each engine execution  
• Prevent skipped engines  
• Log execution status

# 3. Market Intelligence Snapshot Persistence Issue

Another major issue occurs in the Market Intelligence section.

Observed behavior:

• Market analysis runs successfully  
• Results are displayed  
• After leaving the section and returning, the results disappear

This strongly indicates a snapshot persistence issue.

Possible causes:

• Snapshot not saved after analysis  
• Snapshot saved but not retrieved on reload  
• Snapshot cleanup worker removing the snapshot prematurely  
• UI not requesting the snapshot correctly

Required fix:

Ensure the Market Intelligence engine persists its analysis snapshot.

The system must:

• Save analysis results in a persistent store  
• Retrieve snapshot when user re-enters the section  
• Prevent cleanup worker from removing active snapshots  
• Ensure UI fetches the stored snapshot on reload

The analysis result must remain visible unless the user explicitly reruns analysis.
