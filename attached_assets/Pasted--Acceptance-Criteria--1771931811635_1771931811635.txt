A) BACKEND — REQUIRED ENDPOINTS (Expose existing data only)

You must create a backend “Control API surface” that exposes what we already store.

1) Audit Feed API (must be filterable + paginated)

Purpose: A live feed of everything happening in the system.

Requirements:
	•	Filter by:
	•	event_type (single or multiple)
	•	module (strategic-core / ci / leads / publishing / meta / worker / gates)

	•	date range: from → to

	•	limit (default 50, max 200)
	•	cursor or page+offset

2) AI Usage Stats API (weekly budget + breakdown)

Purpose: expose AI usage truth from ai_usage_log.

Required output:
	•	Weekly usage summary:
	•	used_tokens, remaining_tokens, budget_tokens

	•	Per-endpoint breakdown

	•	Must include the most recent AI call:
	•	model, endpoint, tokens, duration, success/failure

Hard rule:
	•	Must show 402 budget exceeded events in the audit feed too (so user sees why things stopped).

3) Safety Gate Status API (truth table)

Purpose: Show what gates are currently blocking execution.

Required gates status:
	•	hasApprovedPlan (true/false + reason)
	•	isPaused (true/false)
	•	hasCampaignContext (true/false, demo vs real)
	•	metaMode (DISCONNECTED/REAL/etc)
	•	capabilities (fb_publishing_enabled, ig_publishing_enabled, insights_enabled)
	•	aiBudgetAvailable (true/false)
	•	executionIdle / executionLock (true/false)
	•	feature flags relevant to current modules

Must return:
	•	gates: [{ gateName, status: PASS|FAIL|WARN, reason, lastCheckedAt }]

4) Decision History API (autonomous decisions)

Expose:
	•	last N decisions (default 20)
	•	decision type, reason, risk_level, confidence_score, createdAt
	•	linked audit events (if possible by ID or time window)
	•	must support filters by campaign/plan when available

5) Publishing History API

Expose:
	•	last N publish attempts
	•	status + timestamps + errorReason
	•	include Meta error classification where available (temporary vs permanent)
	•	must show retry count + backoff state if rate-limited

6) Job Queue Status API

Expose:
	•	all RUNNING + last 20 completed/failed jobs
	•	job type, progress %, startedAt, duration
	•	hard timeout markers (if job auto-failed)
	•	concurrency counters (CI jobs running, Strategic jobs running)
