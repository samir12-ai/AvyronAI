1) Confidence-Aware Completion Must Be Guard-Decision Based (Not Threshold Based)
────────────────────────

Current logic allows entryStrategy / narrativeSynthesis to be null when confidence is below a threshold.

This must be tightened.

New invariant:

If guardDecision === "PROCEED":
    • entryStrategy MUST be present
    • defensiveRisks MUST be present
    • narrativeSynthesis MUST be present

If guardDecision === "DOWNGRADE" or "BLOCK":
    • entryStrategy MAY be null
    • narrativeSynthesis MAY be null
    • defensiveRisks must still exist (even if defensive)

We do NOT rely on numeric thresholds alone.
We rely on final guardDecision outcome.

Add:

• Explicit invariant check inside validateSnapshotCompleteness
• Regression test:
    - guardDecision = PROCEED + entryStrategy = null → must fail validation
    - guardDecision = DOWNGRADE + entryStrategy = null → must pass

This prevents miscomputed confidence from silently weakening structural integrity.

No direct db.insert(miSnapshots) calls allowed anywhere else.

Add regression test:

We eliminate future accidental bypass.
