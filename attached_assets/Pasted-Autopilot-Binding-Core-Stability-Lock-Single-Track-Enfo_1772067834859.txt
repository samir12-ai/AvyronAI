1) P0 — Autopilot Not Bound to Approved Plan (Critical Single-Track Break)

Current issue:

I have a generated plan and it is APPROVED, yet Autopilot is not connected to it.
Autopilot behaves as if no active plan exists.

This breaks the single-track architecture.

Required:
	•	Autopilot must resolve:
Active Plan = latest strategic_plan where status = “APPROVED” for the selected campaign.
	•	Autopilot must be hard-gated:
	•	If NO approved plan → return:
{ error: “BLOCKED_NO_APPROVED_PLAN” }
and produce ZERO AI actions.
	•	If APPROVED plan exists → Autopilot must read:
	•	planId
	•	required_work
	•	calendar_entries
	•	execution progress
	•	performance snapshots
	•	strategy insights
	•	AI actions must reference:
	•	planId
	•	sourceTag
	•	evidenceMetric
	•	evidenceTimeframe
	•	priority + justification

UI Requirements:
	•	Show:
	•	Active Plan ID
	•	Plan status
	•	Autopilot state: CONNECTED / BLOCKED
	•	Reason if blocked
	•	No filler actions. No demo bleed.

Acceptance:
	•	Approved plan → Autopilot CONNECTED and actions reference real plan artifacts.
	•	No approved plan → Autopilot BLOCKED and no actions returned.
	•	Add automated test for both scenarios.
