	1.	Blueprint Versioning (Mandatory)

Every confirmed blueprint must generate:
	•	blueprint_version (incremental per campaign)
	•	created_at
	•	previous_version_reference (if exists)

When a user edits after confirmation:
	•	new version is created
	•	old version remains immutable

All downstream execution (orchestrator outputs, validation results, testing plans) must reference blueprint_version explicitly.

No overwriting.
No silent mutation.

We need traceability.

	2.	Full Audit Logging for Strategic Flow

Log the following events:
	•	GATE_PASSED
	•	EXTRACTION_COMPLETED

	•	BLUEPRINT_CONFIRMED (version number)
	•	BLUEPRINT_RESET
	•	VALIDATION_FAILED
	•	ORCHESTRATOR_GENERATED

Each log must include:
	•	accountId
	•	campaignId
	•	blueprint_version (if exists)
	•	timestamp

We need full replay capability for debugging and optimization.
