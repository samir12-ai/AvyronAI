We need to implement a clean architectural upgrade to fix the core decision inconsistency without adding complexity or new engines.

This is NOT a feature request. This is a system correction to ensure all engines operate on structured intelligence instead of raw interpretation.

---

## 🔴 Core Issue

Currently:
- Audience Engine performs analysis but outputs unstructured interpretation
- Positioning Engine still has room to fallback to generic patterns
- This leads to decision drift (e.g., defaulting to simplicity instead of true root-cause-driven positioning)

---

## 🧠 Required Architecture (MANDATORY)

We need to enforce a Layered Signal Flow:

Market Data (Posts / Hooks)
→ Audience Engine (Multi-layer analysis)
→ Structured Signal Layer
→ Positioning Engine (Selection ONLY)

---

## 🧩 1. Audience Engine Upgrade (Layered Output)

Audience must no longer output paragraphs.

It MUST output structured signals in 3 layers:

### Layer 1 — Surface Signals
- Extracted pains (from hooks)
- Extracted desires
- Dominant themes

### Layer 2 — Pattern Signals
- Repetition clusters
- Frequency-weighted patterns
- Market narrative trends

### Layer 3 — Interpretation Signals (CRITICAL)
- Root causes (derived, not copied)
- Psychological drivers
- Hidden motivations

---

### ✅ Required Output Format

Audience MUST return structured JSON:

{
  "pain_clusters": [...],
  "desire_clusters": [...],
  "pattern_clusters": [...],
  "root_causes": [...],
  "psychological_drivers": [...]
}

No free-text summaries allowed.

---

## 🧩 2. Signal Layer (New Logical Layer — No New Engine)

This is NOT a new service.

This is a data contract enforcement layer:

- All downstream engines MUST consume signals
- No engine is allowed to re-interpret raw market data

---

## 🧩 3. Positioning Engine Restriction (CRITICAL)

Positioning must be converted to:

> Selection Engine — NOT Interpretation Engine

---

### Hard Rule:

Positioning MUST:
- Select from root_causes
- Select from pain_clusters
- Build narrative ONLY from provided signals

---

### Forbidden Behavior:

- Generating new pains
- Using generic patterns (simplicity, ease, innovation…)
- Ignoring signal inputs

---
