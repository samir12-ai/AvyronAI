# 2 — Segment Density Mathematical Error

Segment Density currently returns values such as:

```
67%
74%
5%
```

This violates basic probability rules because total density exceeds 100%.

This means the density algorithm is using **raw signal weights without normalization**.

### Required Fix

Segment density must be normalized.

Correct formula:

```
density = segment_signal_weight / total_signal_weight
```

After calculating density for each segment:

```
normalize densities so total = 100%
```

This guarantees mathematically consistent outputs.

### Required Validation

Add automated test:

```
sum(segment_densities) == 100%
```

If not, the engine must fail validation.
