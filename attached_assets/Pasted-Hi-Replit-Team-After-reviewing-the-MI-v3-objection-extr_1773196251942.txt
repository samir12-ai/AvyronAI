# 1. Core Correction: Comment Counts Must NOT Be Used for Objection Detection

Our platform does not scrape Instagram comment text by design. The system only receives:

- comment_count (numeric)
- no comment_text

Because of this, comment data must be treated strictly as an engagement signal.

Correct rule:

comments_count → engagement signal only  
NOT → objection signal  
NOT → objection confidence input

Important guardrail:

comment_count must not increase OR decrease objection confidence.

It should not enter the objection scoring system at all.

# 3. Separate Pain vs Objection vs Trust Barriers

The system must not mix these categories, which is a common error in market-analysis tools.

Required separation:

Pain signals  
→ market problems

Objections  
→ resistance to buying the solution

Trust barriers  
→ skepticism toward the provider or mechanism

Example:

Pain  
“Leads are inconsistent”

Objection  
“Agencies waste ad budgets”

Trust barrier  
“Agencies overpromise results”

These must remain distinct signals in the pipeline.
